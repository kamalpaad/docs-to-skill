# Expense Reimbursement Guide

1. **Verify Timeline**: Confirm the purchase occurred within the last 30 days, as older submissions are automatically rejected.
2. **Access Expensify**: Instruct the user to log into the Expensify app using company SSO.
3. **Prepare Receipt Documentation**: Advise the user to take a photo of the receipt. If the expense is over $75, explain that an itemized receipt is mandatory and a credit card statement alone is insufficient.
4. **Select Expense Category**: Instruct the user to categorize the expense as 'Travel', 'Meals', 'Software/Subscriptions', 'Office Supplies', or 'Other' (recommend avoiding 'Other' to prevent approval delays).
5. **Apply Specific Expense Policies**:
   - **Meals**: Enforce a $50/day cap for travel meals. For team lunches (no cap), ensure at least 3 attendees are listed.
   - **Client Entertainment**: Confirm that VP approval is attached *before* submission.
   - **International Purchases**: Submit in the local currency; Expensify will auto-convert based on the day-of-purchase rate.
   - **Mileage**: Instruct the user to use the built-in Expensify mileage tracker rather than manual estimation.
6. **Explain the Approval and Payment Flow**: Inform the user that approval goes from manager to finance to payment, taking 5-7 business days. Payment is sent via direct deposit as a separate line item from payroll.
7. **Handle Rejections**: If a claim is rejected, instruct the user to review the email reason code, edit and resubmit the existing claim, and emphasize not opening a new one.
